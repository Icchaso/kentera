# Claude Code 活用アイデア（KENTERA）

KENTERAプロジェクトでClaude Codeを最大限に使いこなすためのアイデア集。

---

## 1. 動画台本の自動生成・改善

### できること
- 「素材/」フォルダ内のYouTube書き起こしを読み込ませ、指定テーマの台本を一発生成
- 既存台本（B03〜B09）の「編集前」状態を、KENTERAのトーン・構成に合わせて自動編集
- 台本の「共感→結論→判断軸→CTA」構成を確認・修正

### 使い方例
```
「素材フォルダの栄養学講座を参考に、B01_血液系の台本を作って。
患者向け動画台本の構成に従って。」
```

---

## 2. LINEステップ配信の文章生成

### できること
- 患者向け・治療家向けそれぞれのLINE配信文を一括生成
- 「ウェルカム→教育→オファー」の流れに沿って全6〜7通分を一気に作る
- A/Bテスト用の別バージョン作成

### 使い方例
```
「患者向けLINE設計.mdを読んで、0通目〜5通目のLINE配信文を全部書いて。
500円購入へ自然につながるように。」
```

---

## 3. 契約院の進捗レポート自動生成

### できること
- `04_契約院管理/院別/` の各院ノートを読み込んで月次サマリーを自動作成
- KPI推移から課題・次のアクションを提案
- 複数院を横断比較して成功パターンを抽出

### 使い方例
```
「院別フォルダの全院を読んで、今月の進捗サマリーと
注意が必要な院を教えて。」
```

---

## 4. ミーティング議事録の整理・要約

### できること
- 長い音声書き起こし（07_ミーティング）を構造化されたノートに変換
- 決定事項・次のアクションを自動抽出
- 関連する他ノートへのリンク付け

### 使い方例
```
「2026-01-08_FANTSキックオフ.md を読んで、
決定事項と次のアクションを整理して。」
```

---

## 5. 測定会スクリプト・台本の作成

### できること
- `03_測定会/` の設計資料から、当日の進行台本・トーク例を生成
- 「結果説明→バックエンド提案」の自然なトーク流れを作成
- 患者タイプ別の対応例を作成

### 使い方例
```
「測定会全体設計.mdとオリゴスキャン測定会の資料を読んで、
当日の進行台本（受付〜成約まで）を作って。」
```

---

## 6. 新機能：Vault全体の俯瞰・整合性チェック

### できること
- `HOME.md` の優先タスクリストと各フォルダの実際の状態を照合
- 「動画構成インデックスに未作成と書いてあるが実際は作成済み」などのズレを検出
- ノート間のリンク切れを発見・修正提案

### 使い方例
```
「Vaultを俯瞰して、HOME.mdのタスクリストと
実際のファイル状況のズレを教えて。」
```

---

## 7. toBコンサル提案書・営業資料の生成

### できること
- 参考素材フォルダのコンテンツ + KENTERA概要から提案書PDFの原稿を生成
- 院別の課題に合わせてカスタマイズした提案書を作成
- ウェビナースクリプトの生成

### 使い方例
```
「参考素材インデックス.mdと事業モデル全体像.mdを読んで、
新規治療院向けのウェビナー台本（30分）を書いて。」
```

---

## 8. テンプレートの自動適用

### できること
- 新しい契約院を追加する際、テンプレートを読み込んで院名・日付を入れたファイルを一発生成
- ミーティングノートを決まった形式で毎回作成

### 使い方例
```
「契約院ページテンプレートを使って、
〇〇整骨院のページを04_契約院管理/院別/に作って。
契約開始日は2026-04-01。」
```

---

## 9. FANTSコンテンツ進捗ダッシュボード更新

### できること
- 動画構成インデックスの「状態」列を実際のファイル存在確認で自動更新
- 「台本あり」「完成」「未作成」を自動判定してテーブルを更新

---

## 10. 重要判断ログへの自動追記

### できること
- 会話の中で決まった方針・判断を `重要判断ログ.md` に自動整理して追記
- 過去の判断を引用して「この方針はいつ決まったか」を即座に確認

---

## Claude Codeとの作業フロー（推奨）

```
1. HOME.md の優先タスクを確認
2. Claude Codeに「今日やること」を伝える
3. 関連ファイルを読み込んで作業実行
4. 生成物をVault内の正しい場所に保存
5. 動画構成インデックスの状態を更新
6. 重要判断があれば判断ログに追記
```
