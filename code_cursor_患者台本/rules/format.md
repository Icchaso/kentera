# 台本フォーマット（3種）

元テキストから生成する3種類のファイルそれぞれの出力形式を定義します。

---

## ① スライド用フォーマット（`出力/スライド/`）

スライド作成ツールに流し込む用。1スライド＝1ブロック。

```markdown
# 3行サマリー
- この動画のゴール：＿＿＿＿
- 視聴後にできること：＿＿＿＿（行動レベル）
- 次の一手（CTA）：＿＿＿＿（1つだけ）

---

# スライド1: （スライドタイトル）
## ナレーション
（読み上げ用テキスト）

---

# スライド2: （スライドタイトル）
## ナレーション
（読み上げ用テキスト）
```

**ルール**
- `# スライドN:` → スライド見出し（スライドタイトルになる）
- `## ナレーション` → そのスライドで読み上げる本文
- `---` → スライド区切り

**見出しのルール（重要）**
- スライドタイトルには**そのスライドの内容を端的に表す見出し**を付ける
- 構成上の役割（「共感フック」「結論先出し」「パラダイムシフト」「不安ゼロ設計」「CTA」「まとめ」等）を見出しに使ってはならない
- 視聴者がタイトルだけ見て「何の話か」がわかる、コンパクトで具体的な表現にする
- 良い例：`# スライド1: 「治る」には2種類ある`、`# スライド2: 今日おさえる3つのポイント`、`# スライド9: 薬はやめていいの？`
- 悪い例：`# スライド1: 共感フック＋パラダイムシフト`、`# スライド2: 結論先出し（今日の3つ）`、`# スライド9: 不安ゼロ設計`

---

## ② 読み上げ台本用フォーマット（`出力/読み上げ台本/`）

AI音声ツール・実際の読み上げ用。**ナレーション本文テキストのみ**を抽出する。マークダウン装飾（#・##・**等）は使わない。スライド区切りなどの記号・ラベルも一切入れない。

```
（スライド1のナレーション本文）

（スライド2のナレーション本文）
```

**ルール**
- **スライド番号・区切りラベル（`【スライドN】`等）は入れない**（音声読み上げ時に読まれてしまうため）
- スライド間は**空行1行**で区切る
- テキスト以外の装飾は一切入れない
- 読み上げツールにそのままコピペするだけで使える、純粋なナレーションテキストにする
- 範囲表現は `〜` を使わず、`から` を使う（例：`夕方4時〜5時` → `夕方4時から5時`）
- **読み間違い防止：読み間違いが起こる可能性のある単語はすべてカタカナ表記にする**
  - **1文字の漢字**は読み違いが起きやすい（例：方→カタ／ホウ、生→セイ／ナマ、波→ナミ／ハ）。複数の読みがある1字は特にカタカナにしやすい（例：下→シタ／サ／クダ）
  - **熟語**は文脈で読みがほぼ決まり、読み違いは起きにくいことが多い。安易にすべてカタカナにする必要はない
  - 熟語でも読みが例外的に紛らわしいもの（例：本編→ホンペン、最終章→サイシュウショウ）
  - 文脈で読みが変わる語（例：根本→コンポン／ネモト、最中→サイチュウ／モナカ、便→ベン／タヨリ）
  - 動詞で読みが紛らわしいもの（例：治まる→オサマル、蓄える→タクワエル、偏る→カタヨル、焦る→アセル）
  - 副詞で読みが崩れやすいもの（例：あくまで→アクマで）
  - 迷ったらカタカナにする。読みやすさ・正確さを最優先する

---

## ③ 要約用フォーマット（`出力/要約/`）

テキストで保管・共有する用。内容を凝縮した構造化テキスト。

```markdown
# タイトル

## 3行サマリー
- ゴール：＿＿＿＿
- 視聴後にできること：＿＿＿＿
- CTA：＿＿＿＿

## 主要ポイント
1. ＿＿＿＿
2. ＿＿＿＿
3. ＿＿＿＿

## 各スライドの要点
- スライド1：＿＿＿＿
- スライド2：＿＿＿＿
（以下同様）

## 今日からのアクション
- ＿＿＿＿
```

**ルール**
- スライド1枚の要点は1〜2行に凝縮する
- 読んで内容が把握できる最小限の情報量にする
- Markdown形式で保存（見出し・箇条書きを活用）
