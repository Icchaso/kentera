# 台本フォーマット（3ファイル出力）

元テキストから生成する3種類のファイルそれぞれのフォーマットを定義します。

---

## ① スライド生成用フォーマット

**使い方：** スライドツール（PowerPoint・Canva等）に貼り付けてスライドを作成する素材ファイル。

`#` をプレゼンタイトル、`##` を1枚のスライドとして扱う。

```markdown
# （動画タイトル）

## （スライド1タイトル）
（このスライドのメインメッセージを1〜2文で。箇条書きの前に置く）

- ポイント①
- ポイント②
- ポイント③

画像：（ビジュアル要素の指示。例：「グラフ、医療、男性、データ」）

## （スライド2タイトル）
（メインメッセージ）

- ポイント①
- ポイント②

画像：（ビジュアル指示）

## （スライド3タイトル）
...
```

### ルール
- `#` → プレゼン全体のタイトル（1つだけ）
- `##` → 1枚のスライド
- `##` 直下の文 → そのスライドのメインメッセージ（1〜2文に絞る）
- `- ` 箇条書き → 補足ポイント。3点以内に絞る
- `画像：` → ビジュアル要素の指示。日本語キーワードで書く
- ナレーション文・長い説明文は含めない（ナレーションファイルに記載）
- メモ・サマリー情報は含めない（要約ファイルに記載）
- **左下に重要情報を置かない**（廣瀬医師のワイプ映像が左下に重なるため、重要なテキスト・数字・図表は左下を避けて配置する）

---

## ② ナレーション専用フォーマット（読み上げテキストのみ）

廣瀬医師の読み上げ専用。マークダウン装飾なし、テキストのみ。

```markdown
【スライド1】
（このスライドで読み上げるナレーション本文をそのまま書く。句読点あり、自然な話し言葉で。）

【スライド2】
（ナレーション本文）

【スライド3】
（ナレーション本文）
```

### ルール
- `【スライドN】` → どのスライドに対応するかの目印（最小限のラベル）
- **見出し（#）・箇条書き（-）・太字（**）は一切使わない**
- 話し言葉（「〜ですね」「〜していきましょう」等）で書く
- 句読点を適切に入れ、自然に読めるようにする
- 1スライドあたりのナレーションは30〜90秒を目安

---

## ③ 要約用フォーマット（FANTS動画概要欄）

FANTSの動画概要欄に掲載するテキスト。用語の予習→動画の価値→ポイント整理→実践→一言メッセージの流れで、視聴前後の両方に使える構成にする。

```markdown
【（動画タイトル）】

〜知っておくと読みやすい用語〜
・（用語）→ （平易な解説）
・（用語）→ （平易な解説）
・（用語）→ （平易な解説）

〜この動画でわかること〜
（動画全体の価値を1文で端的に伝える。「〜がスッキリわかります」等）


【今日の○つのポイント】

【1. （ポイントタイトル）】
・（要点）
・（要点）
（例）（比喩や具体例で補足）
・（要点）

【2. （ポイントタイトル）】
・（要点）
（例）（比喩や具体例）
・（要点）

【3. （ポイントタイトル）】
...


【今日からやること】

・今日 → （すぐできるアクション）
・3日以内 → （短期アクション）
・1週間後 → （次のステップ）


〜ひとことメッセージ〜
（動画の本質を凝縮した一言。前向きで、行動を後押しする表現で締める）

〜参照情報〜（外部リサーチを行った場合のみ記載）
・（参照したソース・URL・論文名など）
```

### ルール
- 用語解説は冒頭に置き、視聴前の予習として機能させる
- 用語解説は「・用語 → 解説」の形式。新卒社会人でもわかる平易な表現にする
- 「この動画でわかること」は1文で端的に書く
- ポイントは動画の核心を番号付きで整理し、（例）で比喩や具体例を添える
- 「今日からやること」は時間軸（今日／3日以内／1週間後）で段階的に示す
- ひとことメッセージは前向きな一言で締める
- マークダウンの見出し（#）は使わない。【】と〜〜で区切る

---

## フォルダ構成

```
writing rules/    ← 台本生成ルール
.cursor/rules/    ← Cursor自動読み込みルール（システムフォルダ。リネーム不可）
outputs/
├── slides/       ① スライド生成用
├── narrations/   ② ナレーション専用
└── summaries/    ③ 要約用
```
